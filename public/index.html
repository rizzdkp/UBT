UBT - Frontend (Static)

Anda sedang melihat versi static dari aplikasi. Aplikasi utama menggunakan Express dan SQLite sehingga membutuhkan backend yang berjalan terus (tidak bisa hanya di Vercel Serverless).

Apa masalahnya?

Vercel menunjukkan 404 karena tidak ada berkas index.html pada root static. Repositori berisi banyak template EJS (views) yang dirender oleh server — sehingga halaman dinamis tidak tersedia di deploy static.

Apa yang bisa dilakukan sekarang

  • Deploy frontend static ke Vercel (sudah): file ini mencegah 404 dan memberi informasi kepada pengunjung.
  • Deploy backend (Express + SQLite + socket.io) ke layanan yang mendukung proses Node panjang dan storage writable, mis. Render, Railway, atau Fly. Setelah backend online, atur environment variable di Vercel: PUBLIC_API_URL → URL backend.

Butuh bantuan?

Jika Anda mau, saya bisa:

  1. Menyiapkan panduan cepat deploy backend ke Render (termasuk opsi persistent disk).
  2. Migrasi sebagian ke serverless (lebih besar pekerjaannya).
